## 〇、一句话结论

**首页 overlay（立即获取免广告特权/剩余X喵）与首页 banner（admobBannerFurtureView）由同一函数 0xbd3540 构建**（字节铁证：slot 0x62ca 立即获取 @0xbd3fbc + slot 0x62cd admobBannerFurtureView @0xbd4228 均在 0xbd3540 内）。而 0xbd3540 由 overlay 编排器 0xbd2e1c 调用。

→ **分析报告8 的 0xbd2e1c SizedBox.shrink patch（0xbd2e24，5 word）一次根除 overlay + banner 双效**（在调用 0xbd3540 之前即返回空，0xbd3540 不执行）。0xbd3540 还加载喵喵页 slot 区的 widget 对象（slot 0x59a4=ref121017），**可能同时是喵喵块的容器**——若主控装机发现块也消失，则 Task1+Task2 一举解决（待验证）。

## 一、字节铁证：0xbd3540 = overlay + banner 双源

`func_slots.py` 列出 0xbd3540（range 0xbd3540-0xbd4ab8）加载的 97 个 slot，其中决定性两条：

| slot | ref | 含义 | 加载点（0xbd3540 内） |
|---|---|---|---|
| 0x62ca | 18033 | "立即获取"（overlay promo 文案） | 0xbd3fbc |
| 0x62cd | 19136 | "admobBannerFurtureView"（banner 视图名） | 0xbd4228 |

→ **0xbd3540 同时构造 overlay 文案与 banner 视图**。它由 0xbd2e1c 的"无特权"分支调用（0xbd2e58: bl 0xbd3540）。

`scan_slot.py` 交叉印证：slot 0x62cd（admobBannerFurtureView）全 app 6 个加载点，0xbd3540 是其中之一（@0xbd4228）。

## 二、★ 强化的 Task2 patch（同分析报告8 §3.2，确认双效）

**0xbd2e1c 入口 SizedBox.shrink 返回**（patch 点 0xbd2e24，5 word）：

```
地址 0xbd2e24 (5 word, 20 字节):
  原始: d1 00 61 ef | f9 40 1f 50 | eb 10 01 ff | 54 00 12 49 | f9 40 0f a0
  patch: 60 73 40 91 | 00 d4 45 f9 | ef 03 1d aa | fd 79 c1 a8 | c0 03 5f d6
        (add x0,x27,#0x1c,lsl12; ldr x0,[x0,#0xba8]=slot0x3975 SizedBox.shrink; mov x15,x29; ldp; ret)
```

**双效机制**：0xbd2e1c 是 Home overlay 编排器，入口即返回 const SizedBox.shrink → 跳过 `tbnz` 特权分支与 `bl 0xbd3540`（overlay+banner 构造）/ `bl 0xbd3084`（已生效版本）→ **overlay 与 banner 均不渲染**。

**为何 patch 0xbd2e1c 而非 0xbd3540**：0xbd2e1c 是上游编排器，patch 它同时跳过 0xbd3540（无特权：overlay+banner）与 0xbd3084（有特权：已生效剩余）两条路径，覆盖最全。直接 patch 0xbd3540 只挡无特权路径。

### 备选（若主控想更精准只挡 overlay+banner 而保留 0xbd3084）
patch 0xbd3540 入口（func+8=0xbd3548）SizedBox 返回（同配方，5 word）。但需主控先核 0xbd3548 原字节。

## 三、喵喵块是否也在 0xbd3540？（待主控验证）

0xbd3540 还加载 **slot 0x59a4（ref121017）@0xbd4188**。slot 0x59a4 位于喵喵页 slot 区（0x59a0-0x59cf，与 ref28525 喵喵饿了/解绑等同区），但 ref121017 是大 object_id（**widget/class 对象，非字符串**），无法静态定名（需 Code 簇）。

**假设**：slot 0x59a4 可能是喵喵块/喵喵组件的 const widget 容器。若然，0xbd2e1c patch 会连带隐藏喵喵块 → **Task1+Task2 一 patch 解决**。

**【需主控 patch-and-test】**：装 0xbd2e1c patch 后，进阅读页观察：
- overlay（剩余X喵/立即获取免广告特权）消失？→ Task2 ✓
- banner（二次元涩兔合集）消失？→ Task2 banner ✓（与 overlay 同源确认）
- 喵喵饿了块消失？→ **Task1 ✓✓（同源根除，意外之喜）**
- app 不崩、阅读正常？→ 安全 ✓

若 overlay+banner 消失但块仍在 → 块不同源（slot 0x59a4 是别的 widget），回退到分析报告8 §4.4 的块候选 B(0xc3def0)/A(二分)。
