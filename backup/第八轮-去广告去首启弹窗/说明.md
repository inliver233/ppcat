## ★★★ 核心困境: 故障弹窗 (未根治, 机制已彻底查明)

### 机制 (logcat铁证 + Dart栈帧)
广告定时器(每~60s)调用: showSplashAd / register(ksad) / initAd / interstitial /
showFullScreenVideoAd / loadFullAd / loadInsertAd / loadSplashScreenAd /
MobileAds#initialize → 因去广告(注释插件)全部抛 **MissingPluginException** →
**未捕获(Unhandled Exception)** → app全局错误处理 → 弹故障窗(ref27673"无法正常联网…广告无法显示").
T+260s必触发, 点确定可关.

### 穷尽失败的方案
1. **NOP各showDialog编排器**(0xbc0f40/0xbd5a24/第三源0xc4cb30/22个广告失败catch函数) → **全无效**.
   证实: 故障窗【不走0x47369c/0x472c98 showDialog BL调用链】! 由全局错误处理异步弹.
2. **AdNoOpPlugin V3 stub**(接管channel, 消除MissingPlugin) → MissingPlugin消失, 但返回值崩溃:
   - success(null) → reward: NoSuchMethod []("channel_name") on null
   - success(空HashMap) → showSplashAd: type HashMap not bool
   - success(Boolean.TRUE) → 下游: type bool not 'qsa?' (期望自定义对象)
   - V3-null 主界面 → Null check operator used on null value (T+66s, 比MissingPlugin更快崩)
   → 不同广告方法期望不同返回类型(bool/Map/自定义对象qsa), 且部分用!强制非null. 统一返回无法满足.

### 根本矛盾
去广告(注释插件)必然MissingPlugin → 故障窗. 消除异常需精确模拟每个广告方法响应(类型各异,
含自定义对象qsa, Java侧无法构造). **这是三分支6轮+第八轮都没根治的核心难点.**

### 已排除的方向
- isNoAdLock总闸0x7e8540 / SP showSplashAd / 0x8863e8开屏NOP: 对可见/WebView广告无效
- 0x913bf8: 是设置聚合器非故障源(撤回)
- 故障body builder=0x920d7c (3 callers: 0x920b3c/0xc4cb30/0xc4d22c, 均无showDialog BL)

### 待调研方向 (交test1/2/3)
1. **定位app全局错误处理**: runZonedGuarded / FlutterError.onError / zone onError 注册点(把未捕获异常转成故障窗的地方). 截断它=所有未捕获异常静默. 风险:掩盖真错误.
2. **逐方法精确V3**: V3按method名返回正确类型. 需解码Pangle(C0984)/GDT/Ksad/AdMob每个方法的Dart端期望返回类型(用Ghidra反编译Dart AOT的invoke回调解析). test1有C0984反编译基础.
3. **重新启用插件+改内部空操作**: 不注释注册(避免MissingPlugin), 改每个插件onMethodCall跳过真实广告加载直接返回原格式响应.
4. **Dart侧NOP广告invoke调用**: 在广告方法async入口(0x888b18/0x8880c0/0x88862c)NOP掉对channel.invoke的调用, 让方法直接返回(不抛异常).
