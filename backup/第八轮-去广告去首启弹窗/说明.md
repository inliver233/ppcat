## 当前版本做了什么 (皮皮喵修改-第八轮.apk, versionCode 1146)

### 2. 反篡改 (libapp)
- 0x8e1dd0 / 0x8ef2b8 入口 return null (篡改检测窗不弹)

### 3. 故障弹窗抑制 (libapp, 第七轮)
- 0xbc1020 故障门控重定向
- 0xbc1058/0xbc1134/0xbc1234 (0xbc0f40 的3个 dialog-show BL) = NOP
- 0xbd60ac/0xbd6264/0xbd6300/0xbd6130 (0xbd5a24 的4个 dialog-show BL) = NOP
→ 故障弹窗主路径(编排器1+2)已堵.

- 0xba0bbc: e00f2036 → 7f000014 (跳过"饿了喵~"提示块)
→ 对喵喵饿了点击行为无实质改变, 保留作无害补丁.

## 第八轮运行时证伪的静态结论 (重要, 供后续参考)
以下 test2/test3 报告的"乐观"方案, 运行时实测无效, 已放弃:
- isNoAdLock 总闸 0x7e8540:69000014 → 对可见广告基本无效
- SP showSplashAd=false → 不阻止 WebView 开屏
- 0x8863e8 开屏控制器 NOP → 不阻止 WebView 开屏 (WebView 是 AdMob SDK 行为, 不走 Dart showDialog)
- 0x913bf8 第三故障编排器 → 已撤回 (是设置聚合器, 非故障源)

## 方法论修正 (解决之前反复出现的"状态污染/装错"bug)
- **install -r 同签名同 versionCode 不替换 libapp.so!** 必须 apktool.yml versionCode 递增才能强制覆盖 native lib.
- **pm clear 清掉书架数据**, 测阅读页需 install -r (保留数据) + versionCode 递增.
- SP 预写要 chown 到 app uid (u0_aNNN) + chmod 660, 否则 app 读不到.
