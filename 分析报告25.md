# 第二十五轮代码恢复收口报告

> 分支：`test1`
>
> 角色：纯静态分析。本轮按用户要求继续吸收 `main / backup / test2 / test3` 的最新成果，重点不再扩散到新工具，而是把 `去广告残留问题 + VIP/特权可用` 收口成更稳的 patch 分层。

## 1. 本轮最重要的纠偏

### 1.1 故障弹窗主方案已稳定，不再摇摆

主控在 `origin/main` 第九轮交接里已经运行时确认：

- `0x920d90: 69 2c 00 54 -> 29 00 00 14`

这条 patch 可以在 `0x920d7c` 的正文构造链入口直接跳过故障正文路径，挂机超过先前必现时间后仍不再弹故障窗。  
因此 `0x920d90` 现在是 `test1` 线的默认主推荐，不再回退到旧的 `showDialog`/catch/NOP 路线。

### 1.2 阅读页 `喵喵饿了` 的旧主线已被主控证伪

来自 `origin/main` 第九轮喵块任务书的运行时事实：

1. `ref28525 "喵喵饿了"` 全局只剩两处直接 LDR：
   - `0xae36fc` in `0xae30e0`
   - `0xa55cbc` in `0xa54178`
2. 主控已实测：
   - 改 `0xae312c` 路径门
   - 改 `0xae32d4` return null
   - 改 `0xa54178` 入口 return null
   - 结果：**阅读页右下角喵块完全没变**

结论：

- `0xae30e0`、`0xa54178` 以及它们的入口级/局部 return-null 路线，**不能再作为阅读页喵块的主推荐方案**。
- 这也意味着我第 24 轮里保留的 `0xae32c8 / 0xae32d4` “尝试不显示”思路，在当前证据下应整体降级，只保留为低优先级实验，不再写成主线。

### 1.3 VIP 默认门控仍应保留 `0x7e861c`

本地重新复核：

```asm
0x007e85f8: bl   #0x7e41c0
0x007e8618: bl   #0x8cf36c
0x007e861c: tbz  w0, #4, #0x7e86f4
```

对应字节：

- `0x7e861c: c0 06 20 36 -> 32 00 00 14`

这条线仍是当前最窄、最像“上层消费侧特权门”的点：

1. 它位于 `0x7e8534` 这个 41 caller 的共享广告/特权 gate 内部。
2. 它直接消费 `0x8cf36c` 返回的 canonical bool。
3. 它比直接粗切 `0x8cf374` 更上层、更轻。

因此本轮继续维持分级：

- `0x7e861c`：默认主推荐
- `0x8cf374`：更广覆盖的实验项
- `0xa54178 / 0xb9fc84`：只做页面/状态机观察，不做入口级主 patch

## 2. 交叉验证后的 patch 分层

### 2.1 稳定纯净栈

这套更适合“去广告 + 不弹故障 + 阅读不被喵块卡住”：

1. `0x920d90: 69 2c 00 54 -> 29 00 00 14`
   - 作用：根治故障正文链
2. `0xbc5dd0..0xbc5dd8`
   - 原：`ef a1 00 d1 a0 0b 40 f9 02 70 41 b8`
   - 改：`e0 03 16 aa fd 79 c1 a8 c0 03 5f d6`
   - 作用：`喵喵饿了` 点击链立即返回 null，块可见但不可点击、不卡加载
3. `0x7e861c: c0 06 20 36 -> 32 00 00 14`
   - 作用：让上层 reward/特权消费门更容易走 true

这套仍是我现在最推荐主控优先保留的组合。

### 2.2 保留 reward/特权效果的实验栈

如果目标偏向“尽量保留特权/喂喵收益”，就不要先上 `0xbc5dd0..0xbc5dd8`，而改试：

1. `0x920d90: 69 2c 00 54 -> 29 00 00 14`
2. `0x7e861c: c0 06 20 36 -> 32 00 00 14`
3. `0x875efc: 40 01 20 37 -> 0a 00 00 14`

本地复核 `0x875efc`：

```asm
0x00875ef4: bl    #0xc833ac
0x00875efc: tbnz  w0, #4, #0x875f24
```

它仍是 `onReward Cheat Triggered!` 分支里的最窄候选，适合只在“保留 reward 路线”时单独实验。

### 2.3 更激进的共享特权态实验

- `0x8cf374: ef 81 01 d1 -> 2a 00 00 14`

本地和 `test3` 一致，这一条仍只适合写成：

- 更底层
- 更共享
- 影响面更大

的实验项，不再把它写成“隐藏喵块主方案”。
