## 三、★ 篡改弹窗真根因 = 签名校验(已抓到完整链)

frida抓到(原生ARM, getPackageInfo + native backtrace):
```
[Java] getPackageInfo("com.gentle.ppcat", flags=0x8000040) — SIG READ
  signatures[0].hashCode=-1414421055  ← 重签名的hash
[Java] MD5 computed ×2               ← 算签名MD5
[Native] 弹窗触发 0x9126b0 HIT        ← MD5不匹配原版→弹篡改
```
- Java层 getPackageInfo(flags=GET_SIGNATURES|GET_SIGNING_CERTIFICATES) 读签名
- 算 MD5(MessageDigest)
- native层(经platform channel)判定: MD5 ≠ 原版硬编码 → 弹篡改
- 弹窗链: 0x914ed4(func 0x913bf8, isCheckCode) → 0x91ca6c → 0x9126b0(弹窗触发) → 0x91280c(正文构造)

## 四、★ 签名伪造绕过成功(runtime验证)

### 原版签名提取
- 从原版APK的 META-INF/PPCAT.RSA (PKCS7) 提取 X509 cert
- CN=Gentle Kwan (原版作者签名), DER 907字节
- 存 `work/orig_x509.der` + hex(`orig_sig_b64.txt`)

### Java hook伪造签名(成功)
```javascript
// frida-java-bridge(frida17需打包, frida-compile)
var Signature = Java.use('android.content.pm.Signature');
var fakeSig = Signature.$new.overload('java.lang.String').call(Signature, ORIG_SIG_HEX); // DER hex
PM.getPackageInfo(...).implementation: ret.signatures.value = [fakeSig]; // 替换为原版签名
```
### 结果
```
[SPOOF] getPackageInfo signatures -> ORIGINAL (hashCode=-1414421055)
篡改=False  ← 弹窗消失!
喵=True     ← 进主界面(4.1MB截图完整UI)
```
**patch版 + 签名伪造 = 平板干净进主界面, 无篡改弹窗。**
