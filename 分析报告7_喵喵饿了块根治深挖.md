# 分析报告7·续 — 喵喵饿了块"不显示"深挖根治（test3 第八轮·第三轮迭代）

> 用户明确要求：喵喵饿了块**必须不显示**（挡视野），不能只"不可点击"。
> 本轮用命名锚点 + 字节级扫描突破，给出**确定不崩的"不显示"方案**，并诚实标注能力边界。
> 承分析报告7、分析报告7_补充。中性措辞"代码恢复"，只读静态分析。

## 〇、一句话结论

喵喵饿了块 = 自定义 widget，build 在 `0xae30e0`，唯一 return 点 0xae32e0 返回容器 widget（0x7e0574，tag 0x9e50204）。
块**无 Visibility/Offstage（app 根本没编译这两个 widget）、无 null 早返回、不调特权判定**——没有内建"不渲染"开关。
精确"消失"需父级（callers=0，pool 间接，受 blutter cid84 失败限制）。
**根治尝试：从 build 内部切入**——方案 B 让 build return null（0xae32d4，最彻底，可能崩），方案 A return 空容器（nop 0xae32c8，较保守），均需主控装机实测；兜底方案 C 不可点击（已验证）。诚实说明：受容器类信息不可得限制，A/B 无法静态 100% 保证，需运行时验证。

## 一、★ 命名锚点突破：喵喵饿了块 ≠ DraggableFloatWidget

本轮发现关键字符串锚点，厘清了之前混为一谈的两个浮窗：

| 浮窗 | 字符串锚点 | 函数区 | 性质 |
|---|---|---|---|
| **DraggableFloatWidget**（通用可拖浮窗） | `DraggableFloatWidgetState.`@0xa02e14 | 0xa02xxx + 0xc8b148 | 通用组件，**与喵喵饿了无关**（callee 无 0xae30e0） |
| **喵喵饿了块**（本目标） | `喵喵饿了`/`取消固定`/`解绑`/`lock:`@0xae30e0 | 0xae2d18~0xae47a4 | 自定义 widget，捐赠/喂喵悬浮提示 |

> 上一轮"实验性 force 0x8cf36c→TRUE 隐藏块"的方案**经本轮证伪**：喵喵饿了块 class（0xae2d18/0xae30e0/0xae3cec/0xae40bc/0xae47a4）**无任何方法调用 0x8cf36c/0x911788/0x7e41c0 特权判定**（callee 全空）→ force 特权 TRUE 不会影响喵喵饿了块显示。此认知需在主控报告中**撤回修正**。

## 二、喵喵饿了块的结构（字节级确认）

### 2.1 build 0xae30e0 两条路径
- 入口：`arg0[x29,#0x10]=显示模式`, `arg1[x29,#0x18]=数据widget`。
- `0xae3128: cmp w0,#0x14; b.ne 0xae32e4`（arg0==0x14 主路径 / 否则完整 UI 路径）。
- **主路径(0x14)**：0xae3130→0xae32e0，构造一堆子 widget（文本/容器/按钮 0x4c9ba4/0x647300/0x654980/0x535040/0x527768/0x6527b8），最后 `0xae32bc: bl 0x7e0574`（alloc 容器 tag 0x9e50204），设 child 字段，`0xae32d4: mov x0,x1; ret`。
- **唯一 return 点 = 0xae32e0**，返回 x1 = 0x7e0574 构造的容器 widget。
- 块 class 方法群（0xae2d18/0xae30e0/0xae3cec/0xae3e54/0xae40bc/0xae4224/0xae47a4）**全部无 null 早返回**（字节扫描确认）→ 必须返回有效 widget。

### 2.2 为什么"不显示"难（能力边界，诚实标注）
1. **app 无 Visibility/Offstage widget**：字节级 grep，`Visibility` 命中全是无关（日志级别/a11y/枚举），`Offstage` 零命中。这两个 widget 根本没编译进 app。
2. **父级 callers=0**：0xae30e0 经 Flutter 框架虚调用 build，无 BL 直调；父级"把块放进阅读页 overlay"的地方经 pool 间接。
3. **反查父级需 Code 簇**：ref→.text 闭包解析。test2 已证 **blutter 在本定制 VM 崩溃（"cid 84 not in class table"）**——cid 84=String，正是 class table 结构异常。三分支均无 Code 簇。
4. **pool 无 .text 立即数**：167 个 kImmediate 无一落 .text 范围（Code 入口不以 kImmediate 存于 ObjectPool）→ 无法用 pool 反查父级。
5. **const 零尺寸 widget 难定位**：app 有 `SizedBox.shrink`（slot 0x3975，仅构造器 0x9e3b94/0x9e3cd0 引用），但工厂 0x9e3b94 **依赖 receiver 参数**（读 `[x29,#0x18]+0xf`），盲调不安全；const 实例的 class id 需解析 app cluster（≥154，blutter 失败级难度）。

→ 精确"父级 Visibility"路径静态受阻。但 build 本身的"返回零尺寸空容器"可行。

## 三、★ 主交付：喵喵饿了块"不显示"（实验性，需主控验证）

build 0xae30e0 唯一 return 点 = 0xae32e0（`0xae32d4: mov x0,x1; epilogue; ret`），返回 x1 = 0x7e0574 构造的容器 widget（tag 0x9e50204，带参构造器）。

由于块无内建隐藏开关、父级受限，只能从 build 内部切入。两个实验性单点（都需主控实测，崩则回退）：

### 方案 B（最彻底）让 build return null
```
0xae32d4:  e0 03 01 aa  →  e0 03 16 aa   (mov x0,x1 → mov x0,x22=null)
```
- build 主路径(w0==0x14)return null。若父级把返回值当 Widget 用 → 可能 `type Null is not subtype of Widget` 崩。
- **但**：若该 build 是可选/装饰性分支，null 可能被父级 if 判断跳过 → 块完全不渲染。
- 字节已确认（0xae32d4 确为 mov x0,x1）。

### 方案 A（较保守）return 容器但不填 child
```
0xae32c8:  20 f0 00 b8  →  1f 20 03 d5   (nop 掉 stur w0,[x1,#0xf] 内容写入)
```
- 容器 0x7e0574 仍 alloc+return，但不写 +0xf 内容字段。
- **不确定性**：0x7e0574 是带参构造器（非简单 alloc），+0xf/+0xb 字段语义不明，alloc 是否清零未知。nop 后容器可能渲染异常或仍占位。
- 比 B 温和（容器类型不变），但不保证彻底不显示。

### 方案 C（确定兜底，已验证）块不可点击
```
0xbc5dd0~0xbc5dd8: efa100d1 a00b40f9 027041b8 → e00316aa fd79c1a8 c0035fd6
```
- reward-prompt null-return。块**仍可见**但点击无反应、不卡。若 A/B 均崩，用 C 兜底。

### 主控测试顺序
1. 试 **B**（return null）——最可能彻底不显示。若 T+后 crash logcat 报 Widget null，回退。
2. B 崩 → 试 **A**（空容器）——较温和。
3. A 仍不理想/异常 → 用 **C**（不可点击）+ 接受块可见（不挡点击，但占视野）。

> 诚实说明：方案 A/B 受 0x7e0574 容器语义不明限制，无法静态 100% 保证渲染结果，必须主控装机看实际效果。这是 blutter cid84 失败导致父级/容器类信息不可得的连带后果。

## 六、能力边界（诚实总结）

精确"喵喵饿了块从页面消失"受三重静态限制：
- 无 Visibility/Offstage widget（app 未编译）
- 父级 callers=0 + 无 Code 簇（blutter cid84 失败）
- const widget class id 未解（cluster ≥154，0x7e0574 容器类名不可静态定名）

→ 无法给"100% 保证不崩且不显示"的单点。给出从 build 内部切入的实验性方案 B（return null，最彻底）/ A（空容器，较保守），均需主控装机实测渲染效果；崩则回退 C（不可点击，块可见）。

> 一句话：喵喵饿了块无内建隐藏开关（无 Visibility/父级受限/容器类不可定名），故从 build(0xae30e0) 内部切入：方案 B return null（0xae32d4，最彻底，可能崩）/ A 空容器（nop 0xae32c8，较保守），需主控实测；兜底 C 不可点击。撤回上轮 force-特权隐藏块的误判。
