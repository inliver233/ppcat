## 二、任务③结果: obj44988 on-disk装机 **崩溃(abort)**

### 2.1 patch配置(装机字节验证全对)
```python
P_OBJ44988_V2 = [(0xa52928,604f5df9),(0xa5292c,ef031daa),(0xa52930,fd79c1a8),(0xa52934,c0035fd6)]
# 基线testC_ovnull + P_OBJ44988_V2. APK内字节验证: 0xa52928=604f5df9... ✓
```

### 2.2 装机结果: **崩溃**
- 启动后皮皮喵秒退, signal 6 (Aborted), 非SIGSEGV
- 崩溃栈(全部在libapp/libflutter):
  ```
  #22 libapp 0xf1dfe8  #23 0x4690a0  #24 0x468fc0  #25 0xf19f4c
  #26 0xf13350  #27 0x468a8c  #28 0xf97074  #29 0x4b5e88  #30 0x4b5bc4
  #31 0x4b5b2c  #32 0xf97074  #33 0x49d39c  #34 0x462410
  #35 libflutter 0x637d7c  #36 anonymous
  ```
- **是 Dart 主动 abort**(类型断言/异常 unhandled), 非立即段错误. obj44988(cid≠0x85e)在某处被按cat类型访问→类型不符→Dart异常→abort
- **结论**: obj44988方案(返回异cid)在redroid原生下**会崩**, 印证之前V5警告(cid敏感). **obj44988不可用**

### 2.3 反篡改候选A单独不够
- 反篡改A-only版(NOP 0x59a1e4/0x59a2d8/0x59a3bc + 基线0x8e1dd0/0x8ef2b8): **反篡改弹窗仍在**("非法篡改/下载")
- 即聚合3点NOP + 0x8e1dd0 entry-null 都不够覆盖redroid检测
- spawn frida早注入(0x59a1e0等force-pass)能抑制(报告13抓backtrace时没弹), 但on-disk候选A不行
- **redroid反篡改需 spawn早注入 或 候选A+候选B(弹窗构造entry-null) 都加**, 待测
