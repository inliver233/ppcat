## 一、★ 本轮关键纠正（反驳 test1 + 自纠）

### 1.1 G+0xee0 = 免广告特权 isNoAdLock，【不是】daily/喂猫字段

这是本轮最重要的纠正，证伪了 test1 报告31-33 的核心假设。

**铁证（L3/L4 双法 backward-slice + full-function backtrace）：**
- 全 .text **25 个 G+0xee0 读取者**，无一在 cat/widget 模块（0xa5-0xbc）
- 全部是 http/广告/日志/功能门控：
  - `0x7e8534` = isNoAdLock getter（串 isNoAdLock/adNum/才能使用！），被 41 处调用
  - hwMd5 反篡改、getCookie 网络、glog 日志
- `0xa56c34` 写 G+0xee0=TRUE 是"每日奖励流程**附带授予免广告特权**"的副作用，不是 daily-complete 状态

**test1 误判链：** test1 看到 0xa56be4（喵喵喂饱了~）→ 0xa56c34 写 G+0xee0，就推断 G+0xee0=daily/喂猫态。但 0xa56c34 的 25 个读取者全是广告函数——**写入侧巧合，读取侧证伪**。

### 1.2 wf_daily L1 候选 0xadcad0 = 误报

L1 提 0xadcad0（读 G+0xee0+猫饿 field@0x33）为隐藏门候选。**我亲自核对证伪**：0xadcad0 串是 `webBgColor/重试联网/tips/获取来源/无法更改设置应用已被非法篡改/忽略`——是反篡改+广告来源+联网重试函数，callees 全是 0x4xxxxx-0x5xxxxx 网络层，**与喵喵块无关**。又一个候选级陷阱。

### 1.3 test1 推崇的 0x846 链 = 证伪（L4 高置信）

L4 字节级证伪：0x846b1c 是**双精度浮点计算**（fdiv/fcmp/fneg），返回 null 或 double，根本不是 widget 构造器；0x846f30 读 per-object param.field@0x47 做枚举判断，不读全局 daily 状态。整段 0x846000-0x847400 **无任何 BL 到 cat builder 0xa52920/0xa52bb0**。→ 0x846 链与悬浮块**无直接构造关系**。

### 1.4 reward 写的所有状态字段，读取者都不在喵喵块路径（L5 高置信）

L5 穷举 reward 写入的全部字段读取者：

## 三、★ 唯一主推：obj44988 返回（未验证高潜力）

经过 19 agent 彻底排查，**所有其他路径全死，只剩 obj44988 一条还活着且未用正确字节测过**。

### 3.1 为何是它

1. **app 原生"渲染空"widget**：obj44988(slot0x753)在 `0x546760`"尺寸==0"分支返回，231 处复用，**确定非 String**（strings+names双缺）。
2. **字节已验证正确**：slot0x753 byteoff=**0x3a98**（0x753×8），`ldr x0,[x27,#0x3a98]`=**604f5df9**（capstone验证）。
3. **同类槽已证不崩**：主控测过 obj51369(slot0x1db)/obj91683(slot0x368d) 返回→**空白不崩**，证明该 Stack 槽是 Widget 多态（接受任意 Widget）。
4. **从未用正确字节测过**：主控文档偏移写 0xea8（=ref964，错对象），所以"obj44988 渲染不稳"测的根本不是 obj44988。
5. **比已测的两个更强**：obj44988 是 app 对**这类槽的原生空 widget**，比 obj51369/obj91683 更可能"真消失"而非"留空白"。

### 3.2 patch（两方案）

**方案①（覆写 builder 入口，推荐）**：
```
地址 0xa52bb8 (func+8, 16字节覆写), 保留序言 0xa52bb0-0xa52bb7:
  原: ef 41 00 d1 | 50 1f 40 f9 | ff 01 10 eb | a9 05 00 54
  改: 60 4f 5d f9 | ef 03 1d aa | fd 79 c1 a8 | c0 03 5f d6
      (ldr x0,[x27,#0x3a98]=obj44988; mov x15,x29; ldp x29,x30,[x15],#0x10; ret)
```

**方案②（单字替换 tail-call，更彻底，使真 builder 成死代码）**：
```
地址 0xa52b94 (1字替换):
  原: 07 00 00 94   bl 0xa52bb0
  改: 60 4f 5d f9   ldr x0,[x27,#0x3a98]=obj44988
  (0xa52b90 已 push 8字节, 0xa52b98 add x15,#8 配平, 0xa52b9c mov x15,x29, ldp/ret 返回 obj44988)
```

### 3.3 装机验证步骤（主控）
1. apktool.yml versionCode 递增（强制换 libapp）。
2. 装上述 patch（方案①或②）。
3. 进阅读页观察喵喵块 [720,1260][1080,1392]：

   - **崩** → 回退兜底（0xbc5dd0 块可见但不可点击，已证安全）

## 四、诚实边界

1. obj44988 方案使块**恒空**，不等价 ground truth 的"当日隐藏"——但这是静态能做到的最接近"看不到"的方案。
2. obj44988 视觉结果（消失 vs 留空白）**只能装机确认**，静态无法保证。
3. 若 obj44988 也只留空白，则喵喵块"完全不可见"的静态解**确实到顶**——只能接受兜底（可见但不可点击）。
4. 真正复现"当日隐藏"需动态 hook reader build（主控 frida 不可用）或 test1 突破 cid84 解出 Code 对象图。
