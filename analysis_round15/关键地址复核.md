# 第十五轮关键地址复核

> 来源：`test1` 本地对原始 `libapp.so` 的 `capstone` 复核。
>
> 目标：把本轮报告里要继续依赖的关键地址，收敛成最小证据集。

## 1. `0x7e8534` 广告总闸

原字节：

- `0x7e8540 = 50 1f 40 f9`

关键片段：

```asm
007e85dc: 6ba40494    bl      #0x911788
007e85f0: a0835ff8    ldur    x0, [x29, #-8]
007e85f4: 00082036    tbz     w0, #4, #0x7e86f4
007e85f8: f2eeff97    bl      #0x7e41c0
007e8614: 64db55f9    ldr     x4, [x27, #0x2bb0]
007e8618: 559b0394    bl      #0x8cf36c
007e861c: c0062036    tbz     w0, #4, #0x7e86f4
007e86e4: c0c20091    add     x0, x22, #0x30
007e86e8: ef031daa    mov     x15, x29
007e86ec: fd79c1a8    ldp     x29, x30, [x15], #0x10
007e86f0: c0035fd6    ret
```

结论：

- `0x7e86e4` 是 canonical `True` 尾声
- `0x7e8540 -> 69 00 00 14` 的静态语义成立：直接跳转到 `True` 返回
- `0x7e85f4 -> 07 00 00 14` 也仍是有效备选

## 2. `0xba0bbc` reward / 饿了喵块

原字节：

- `0xba0bbc = e0 0f 20 36`

关键片段：

```asm
00ba0bb8: a0035ef8    ldur    x0, [x29, #-0x20]
00ba0bbc: e00f2036    tbz     w0, #4, #0xba0db8
00ba0bc0: 2051e697    bl      #0x535040
00ba0bc4: a0031cf8    stur    x0, [x29, #-0x40]
```

结论：

- `0xba0bbc -> 7f 00 00 14` 的 patch 语义明确：无条件跳过这段 reward 提示分支

## 3. `0x911788` 共享门控

关键片段：

```asm
009117a0: 404340f9    ldr     x0, [x26, #0x80]
009117a4: 008448f9    ldr     x0, [x0, #0x1108]
009117c0: 01104078    ldurh   w1, [x0, #1]
009117dc: c0033fd6    blr     x30
00911820: f001bfa9    stp     x16, x0, [x15, #-0x10]!
00911830: c0033fd6    blr     x30
```

结论：

- 这是强依赖 live Dart heap / 动态派发的共享门控器
- 它不是一个干净的“小 bool 函数”
- 不建议把它当成单点 return-true patch 位

## 4. `0x8cf36c` 奖励特权状态对象

关键片段：

```asm
008cf3b4: c0c20091    add     x0, x22, #0x30
008cf3b8: a0831ff8    stur    x0, [x29, #-8]
...
008cf41c: c0c20091    add     x0, x22, #0x30
008cf420: ef031daa    mov     x15, x29
008cf424: fd79c1a8    ldp     x29, x30, [x15], #0x10
008cf428: c0035fd6    ret
```

结论：

- 它确实能直接产出 canonical `True`
- 且被 `0x7e8534` 总闸显式调用
- 继续应理解成 reward / no-ad / 特权状态的一层

## 5. `0x8ba3d0` 过期链

关键片段：

```asm
008ba458: 207041b8    ldur    w0, [x1, #0x17]
008ba45c: 00801c8b    add     x0, x0, x28, lsl #32
008ba468: 501f40f9    ldr     x16, [x26, #0x38]
008ba474: 04f040b8    ldur    w4, [x0, #0xf]
008ba47c: c42d0035    cbnz    w4, #0x8baa34
```

结论：

- 这更像日期 / 对象 / 状态结构路径，而不是“单点 isVip 比较”
- 如果后续继续攻 VIP，这里比 `0xb9fc84` 入口更值得单独拆
