## 三、★ 完整性校验(应用保护)根因+修复现状

### 根因(已完全查清)
- 应用自带 `com.gentle.ppcat.hook.PmsHook` 完整性适配框架(App.smali: attachBaseContext调install, onCreate调hookMPM+hookLoadedApk)
- PmsHook.install: 反射 ActivityThread.sPackageManager + Proxy 注入, cert硬编码(原版Gentle Kwan, 正确)
- PmsHook.invoke: 拦截getPackageInfo, 替换signatures=[origSig] + 调makeSigningInfo适配signingInfo
- **Android14失效点**: makeSigningInfo用旧内部类 `PackageParser$SigningDetails` 反射构造SigningInfo → Android14该接口变更 → 抛异常 → catch返回null → signingInfo=null → 完整性校验读到异常 → 提示
- **这就是旧系统反射有效、Android14失效的最终原因**

### 已做修复(PmsHook.smali makeSigningInfo)
- 弃反射, 改用Android14 public接口: `new SigningDetails(sig[],2); new SigningInfo(sd)` (类名 android.content.pm.SigningDetails)
- 验证: signature spoof installed + mPM hooked 成功, 校验提示消失
- **但hookLoadedApk仍失败(No field mPackageInfo in LoadedApk), 修复版渲染异常**

### 校验链(分析得到)
```
Java: getPackageInfo("com.gentle.ppcat", flags=0x8000040) 读取签名信息
  → MessageDigest 计算摘要
  → native(libapp) 经platform channel比对
  → 不匹配 → 0x913bf8(isCheckCode) → 0x91ca6c → 0x9126b0(提示触发) → 0x91280c(内容构造)
```
- 原版签名 hashCode=-1414421055 (Gentle Kwan)
- 运行时适配(frida Signature替换getPackageInfo返回)成功(frida-java-bridge, frida17需frida-compile打包)

### ★ 黑屏根因已查明(2026-06-22 最终诊断): 360加固 + apktool重打包破坏

**原版是 360 加固(jiagu)**:
- `assets/libjiagu.so`, `libjiagu_a64.so`, `libjiagu_x64.so` = 360加固运行时
- `libzeus_direct_dex.so` = 动态加载dex
- 原版 `classes.dex`(8.9MB) 是**加固壳**, 真实业务代码(GDT广告插件注册等)被加密在 libjiagu 内, 运行时动态解密
- 原版 classes.dex 里**没有** PmsHook/gdt_plugins 字符串(壳里没真实代码)

**黑屏机制**:
- apktool 重打包 → 重新 smali→dex → **破坏加固壳结构**(classes.dex 被拆成6个, classes4/5/6只有140字节空dex)
- 加固运行时(libjiagu)加载/校验失败 → 真实代码(GDT插件等)无法解密 → `MissingPluginException(splash on channel plugins.hetian.me/gdt_plugins)` → Dart未捕获异常 → Flutter渲染中断 → 黑屏
- **LDPlayer上能用**: 模拟器上加固可能降级运行(或houdini翻译让校验宽松); 平板Android14上加固严格, 重打包后失效

**结论: 不能用 apktool 重打包原版APK(破坏360加固)**。这推翻了之前所有基于 apktool b 的交付路径(testC_ovnull等在平板上都黑屏, LDPlayer上能用是加固降级的假象)。

**PmsHook是项目自己加的**(不在原版dex), build/smali/com/gentle/ppcat/hook/PmsHook.smali — 但它需要重打包才能进dex, 而重打包破坏加固。矛盾。

### 待新对话解决(正确路径)
**核心矛盾**: 改 dex(PmsHook) 需重打包 → 破坏加固 → 黑屏。可行的路径:
1. **脱壳后再重打包**: 先脱壳(还原真实classes.dex), 再加PmsHook重打包(加固层可去掉或重加固)。需要360脱壳工具(blackdex/fart/frida脱壳)。平板frida可用,可动态脱壳。
2. **只替换libapp.so, 不重打包dex**: 用某种方式(如Xposed/LSPatch/磁力魔方等免重打包注入)替换libapp + 注入PmsHook, 不碰dex/加固壳。LSPatch可免重打包注入dex代码。
3. **加固壳兼容**: 让apktool重打包后仍能过加固校验(难, 需理解libjiagu校验逻辑)。
4. **运行时frida常驻**: gadget脚本模式, 开机自动跑签名适配+libapp内存patch(不重打包)。但gadget会被检测,且sing+0x20翻转触发校验。
- **建议优先LSPatch**(免重打包注入, 不破坏加固) 或 **脱壳后重打包**。
