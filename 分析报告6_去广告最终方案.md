## 二、★ 推荐去广告方案（本机可直接执行）

### 步骤 1：禁阅读期广告（libapp patch）
```
地址 0x7e8540:  50 1f 40 f9  →  69 00 00 14   (B 0x7e86e4 = 强制 isNoAdLock 门闸 True)
```
- **覆盖验证**：0x7e8534 的 41 caller 中 **21 个直接广告相关 + 5 阅读功能 + 13 广告引擎内部**（efc/QMb/ePb 等），全部是"此功能需要看广告/广告无法显示"类决策。强制 True → 41 处全跳过广告装配。
- **安全验证**：41 caller **无一在开屏区 0x7e1000-0x7e3000**，0x7e1464 非其 caller → **不破坏开屏**（关键教训#8）。
- **栈平衡**：跳到 True 尾声 `add x0,x22,#0x30; mov x15,x29; ldp; ret`，prologue 已建帧。
- **备用**：若装机发现广告反增，改 `6d 00 00 14`（强制 False，语义反转保险）。

### 步骤 2：禁开屏广告（SP，零 libapp 风险）
```
adb shell run-as com.gentle.ppcat ...  或 root 后改 SharedPreferences：
  showSplashAd = false
```
- **READ 验证**：0x8863e8 调 getAll(0x4ea988) 读取含 showSplashAd 的 SP → 设 false 生效。
- 比 NOP 0x7e1464 内部 BL **安全得多**（NOP 破坏开屏关闭）。

### 步骤 3：既有兜底（保持）
- 篡改窗 0x8e1dd0/0x8ef2b8 → e00316aac0035fd6
- 故障窗 0xbc1058/0xbc1134/0xbc1234/0xbd60ac/0xbd6264/0xbd6300/0xbd6130 → 1f2003d5
- 更新窗 0x8667a8/0x8675bc → 1f2003d5
- 任务D V1 no-op stub（广告 channel 不抛 MissingPluginException，兜底）

   - **下一步**：或做 EventChannel StreamHandler stub（codec 待解），或动态层（主控本机劫持）。

2. **VIP 身份层**：捐赠绑定(hwMd5→服务器签 remoteConfigSign→rule.bin)，密码学硬问题，无干净静态 patch。
