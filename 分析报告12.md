### 1.2 ★ redroid跑通的关键（你朋友的方案，验证有效）

redroid 14 init.rc硬挂binderfs，但Debian内核无binderfs(CONFIG_ANDROID_BINDERFS not set)。**解决=静态binder设备 + bind mount（不是--device，不是binderfs）**：

```bash
modprobe binder_linux devices="binder,hwbinder,vndbinder"
mkdir -p /dev/binderfs  # mknod /dev/binder 若删了
docker run -itd --privileged \
  --memory=4g --memory-swap=4g --cpus="3" \
  -v /dev/binder:/dev/binder \
  -v /dev/hwbinder:/dev/hwbinder \
  -v /dev/vndbinder:/dev/vndbinder \
  -v /root/redroid-data:/data \
  -p 5555:5555 \
  redroid/redroid:14.0.0-latest \
  androidboot.redroid_gpu_mode=guest \
  androidboot.use_memfd=1 \
  ro.product.brand=Xiaomi ro.product.manufacturer=Xiaomi ro.product.model=M2012K11C \
  ro.product.device=alioth \
  ro.build.fingerprint=Xiaomi/alioth/alioth:13/RKQ1.211001.001/V14.0.6.0.TKHCNXM:user/release-keys \
  ro.build.tags=release-keys
```

**关键点（血泪总结）**：
1. `-v /dev/binder:/dev/binder`（bind mount）**不是** `--device`，也不是依赖binderfs。--device直通binder ioctl走不通，servicemanager死循环刷屏→卡死VM。
2. `androidboot.use_memfd=1` 必须（6.12内核无ashmem，否则vold崩）。
3. **不要设 `ro.secure=1 ro.debuggable=0`**（会关root adb，su失效）。反篡改靠原版APK签名通过，不需关debuggable。
4. `--memory=4g` 兜底（防servicemanager异常时卡死宿主，Oracle免费实例稳定但redroid异常会拖垮guest）。
5. **redroid容器重建后需重注adb公钥**：`docker exec ... echo $PUBKEY > /data/misc/adb/adb_keys`（容器新/data无adb_keys）。

## 五、技术债/教训（记录，避免重走）

1. **redroid静态binder**：必须bind mount(-v)，不能--device，不能依赖binderfs(Debian内核无)。servicemanager死循环刷屏会拖垮VM guest(SSH断/探针离线/控制台在线=guest假死)。
2. **Oracle免费ARM实例稳定**（连续跑2月+），死机是redroid异常导致，非实例本身。
3. **ro.secure=1/ro.debuggable=0会关root**——frida需要root，不能为反篡改关debuggable。反篡改靠原版签名。
4. **原版首启广告(kwad)**是redroid操作的主要障碍，patch版去广告但反篡改触发。
5. **Flutter无semantics**→uiautomator盲，需截图+像素分析或主控导航指导。
6. **frida spawn模式需gadget**（jailed），用attach模式（root下OK）。
