### 2.1 铁证：全"喵喵"字符串仅由 cat 模块加载

## 三、★★ Task2：首页 overlay（剩余X喵 / 立即获取免广告特权）确定性根除

### 3.1 定位

overlay 编排器 `0xbd2e1c`（callers=0，bool_ret=True，616 字节）：

```
0xbd2e34: ldr x0,[x29,#0x18]       ; arg1
0xbd2e38: ldur w1,[x0,#0xb]
0xbd2e48: ldur w2,[x1,#0x13]       ; 特权状态字段
0xbd2e50: tbnz w2,#4,0xbd2e64      ; bit4=TRUE(有免广告特权) → 跳 0xbd2e64
0xbd2e58: bl 0xbd3540              ; 无特权：建"立即获取免广告特权"promo
0xbd2e68: bl 0xbd3084              ; 有特权：建"已生效！剩余X喵"
0xbd2e74: bl 0x654cd8              ; 包裹 widget
```

即：0xbd2e1c 返回这个 overlay widget，由首页 build 挂载。**null/空化它的返回 = overlay 不渲染。**

### 3.2 ★ patch：返回 const SizedBox.shrink（确定不崩）

主控已实证（0xa54178 entry-return-null 测试）：build 入口返回在**本 app 不崩**。但 null 可能触发父级 ErrorWidget。更干净：返回 const SizedBox.shrink（一个合法的空 Widget，任何父级接受，渲染为不可见零尺寸）。

`SizedBox.shrink` const 实例 = **pool slot 0x3975**（ref 1973，type 0 kTaggedObject，已验证）。从 PP(X27) 加载它需 `add x27,#0x1c,lsl12; ldr [#,0xba8]`（slot 0x3975 byteoff = 0x1cba8）。

**通用"返回 SizedBox.shrink"配方**（func+8 起 5 word，保留 prologue 的 STP+mov x29）：

```
func+8:  60 73 40 91   add  x0, x27, #0x1c, lsl #12
func+12: 00 d4 45 f9   ldr  x0, [x0, #0xba8]      ; x0 = slot 0x3975 = const SizedBox.shrink
func+16: ef 03 1d aa   mov  x15, x29
func+20: fd 79 c1 a8   ldp  x29, x30, [x15], #0x10
func+24: c0 03 5f d6   ret
```

（capstone 已验证：`add x0,x27,#0x1c,lsl12; ldr x0,[x0,#0xba8]` → 偏移 0x1cba8 = slot 0x3975 ✓）

**应用于 overlay 0xbd2e1c**（patch 点 0xbd2e24 = func+8，覆盖 5 word）：

```
地址 0xbd2e24 (5 word, 20 字节):
  原始: d1 00 61 ef | f9 40 1f 50 | eb 10 01 ff | 54 00 12 49 | f9 40 0f a0
        (sub x15,#0x18; ldr x16,[x26,#0x38]; cmp x15,x16; b.ls 0xbd3078; ldr x0,[x29,#0x18])
  patch: 60 73 40 91 | 00 d4 45 f9 | ef 03 1d aa | fd 79 c1 a8 | c0 03 5f d6
        (add x0,x27,#0x1c,lsl12; ldr x0,[x0,#0xba8]=SizedBox.shrink; mov x15,x29; ldp; ret)
```

**安全验证（强）**：
1. **栈平衡**：prologue `stp x29,x30,[x15,#-0x10]!`（x15−=0x10）+ `mov x29,x15` 保留；patch 跳过 `sub x15,#0x18`（立即返回不需该帧），尾声 `mov x15,x29; ldp…ret` 把 x15 复位到 x29（stp 位）再 ldp 弹出，完全对称。
2. **类型安全**：返回 const SizedBox.shrink（合法 Widget）。首页 build 把它当 overlay 槽位 → 渲染零尺寸不可见 → overlay 消失。比 null 安全（不会 ErrorWidget）。
3. **隔离**：0xbd2e1c 仅服务于首页特权 overlay，与阅读/开屏/故障无关。
4. **与 0x920d90 正交**：互不影响。

### 3.3 备选：entry-return-null（更简单，3 word）

若主控想要最小改动，或 SizedBox 方案异常，用主控已验证安全的 entry-null：

```
0xbd2e24 (3 word):
  原始: d1 00 61 ef | f9 40 1f 50 | eb 10 01 ff
  patch: e0 03 16 aa | fd 79 c1 a8 | c0 03 5f d6
        (mov x0,x22=null; ldp; ret)
```

主控 0xa54178 实测 entry-null 不崩，但 overlay 槽位可能显示 ErrorWidget（红框）或空。**优先用 3.2 的 SizedBox 方案。**

### 3.4 首页 banner（二次元涩兔合集 + 关闭喵）

- "二次元涩兔合集XXXXX期" 文案**不在 unflutter_strings** → 远程推送/图片配置，非本地 build 文案。
- "关闭喵"（ref21751）由 `0xbd5330` 处理（小 handler：加载"关闭喵"串 + call slot 0x62d6 + return null）—— 这是 banner 关闭按钮的回调，非 banner 容器 build。
- banner 容器是浮窗系统的一个实例（与 overlay 同属 0xbdxxxx/0xc3xxxx 浮窗管理体系）。
- **现状**：banner 内容远程驱动，静态无文案锚点。若主控测 3.2 的 overlay patch 后 banner 也消失，则 banner 与 overlay 共享浮窗容器（一次根除）；若 banner 仍在，需单独定位其容器（建议主控 a11y dump 首页 banner 节点的 bounds/content-desc 反查）。

## 四、★ Task1：喵喵块（最高优先）诚实诊断

### 4.1 根因（本轮定性）

喵喵浮块 = 阅读页 reader 主 build `0xb799b8`（13252 字节）Stack 内的一个**浮块子 widget**。其"喵喵饿了"label：
- 经快照 **const 回引**（const Text/Semantics widget 在快照构建期绑定 ref28525），**运行时无 pool LDR** → scan_slot 在 reader 侧抓不到任何 ref28525 加载点（全 app 仅 cat 区 2 点，均排除）。
- 块本身经 Flutter 框架**虚分派**调用（callers=0，非 BL 直调），父级"把块放进 reader Stack"经 pool 间接。

→ 精确定位"块挂载点"需 Code 簇解析（ref→.text 闭包）。**test2 已证 blutter 在本定制 VM 崩溃（cid84=String，class table 异常），三分支均无 Code 簇。** 本轮我评估了自建 Code 簇解析器：需全簇反序列化（blutter 级难度），且本 VM class table 结构异常致 blutter 失败，自建成功率低、成本高。诚实标注：**这是静态分析的天花板**。

### 4.2 PROVEN 排除（铁证，三方勿重复）

- **0xae30e0**（喵喵菜单索引工厂）：主控 NOP b.ne + null PATH_A → 块不变。本轮确认它是 cat 模块 menu 工厂，非浮块。
- **0xa54178**（喵喵养成/VIP 整页 build）：主控 entry-null → 块还在。本轮确认它加载 cat 区 34 个 slot（0x59a0-0x59c6），非 reader。
- **skmm/yhmm/kamm 图**：主控透明化 → 块不变（块是文字 Button，不用这些图）。
- **特权状态机 0x8cf36c**：主控 force-TRUE → 块不变（块可见性不绑特权）。
- **cat 区全部函数**（0xa56be4/0xb68234/0xbc5dc8）：喵喵串隔离于此，与 reader 浮块无关。

### 4.3 reader build 锚点（给主控/其他分支的定位线索）

- reader toolbar build = **0xb73020**（加载 0x99c1-0x99dd：上一章/下一章/翻页/目录，callers=0）。
- reader 主 build = **0xb799b8**（13252 字节，254 slot，callers=0，bool）—— 阅读页 Stack 编排者。**喵喵浮块是其 Stack 内一个 const-backed 子 widget**。
- 浮块 onTap → ksad channel register → MissingPlugin（主控已证）→ 卡"正在加载"。

### 4.4 最佳 patch（实验性，需主控 patch-and-test）

由于块挂载点静态不可精确定位，给主控**装机实测的候选 + 通用配方**：

**候选 A（最可能）**：浮块是 reader build 0xb799b8 内的 const 子 widget。主控可用 a11y 节点树配合，在 0xb799b8 范围内二分定位块挂载的子 build（块 onTap 闭包的 alloc 点）。一旦定位块容器 build，应用第三节 3.2 的 SizedBox.shrink 配方（func+8 起 5 word）。

**候选 B**：浮窗内容选择器 `0xc3def0`（callers=0，按 arg0 选择器 bl 0xa02e14 建浮窗实例，child 取 pool slot 0xbef/0xbf0）。若浮块是 DraggableFloatWidget 实例，null/空化 0xc3def0 的主路径（0xc3df84 bl 0xa02e14 之后）可能隐藏浮块。但主控 a11y 显示块是 Button（未确认可拖），且 0xc3def0 child 是 icon——置信度中。
```
0xc3def0 entry-return-null (func+8 = 0xc3def8, 3 word):
  原始: d1 00 61 ef | f9 40 1f 50 | eb 10 01 ff  (需主控核对本机字节)
  patch: e0 03 16 aa | fd 79 c1 a8 | c0 03 5f d6
```
注：0xc3def0 是选择器（其它 arg0 已 return null），entry-null 仅影响其主路径浮窗实例。

**兜底 C（块可见但不可点击）**：reward-prompt `0xbc5dc8`（func+8=0xbc5dd0）entry-null（上轮已给，主控证明安全）——块仍可见但点击无反应不卡。不满足"看不到"，但保底。

### 4.5 能力边界（诚实）

精确"喵喵块从阅读页消失"受**三重静态限制**（与上轮一致，本轮进一步确认）：
1. **const-label 墙**：块 label 经快照 const 回引，scan_slot 抓不到 reader 侧加载点。
2. **Code 簇墙**：块 build 经虚分派（callers=0），父级定位需 Code 簇——blutter cid84 失败，三分支都没有。
3. **13252 字节 reader build**：块是其 Stack 内一个子 widget，在 254 个 slot 中无法静态区分哪个是块的 const 容器（需 cluster 解析 const widget class id）。

→ 无法给"100% 静态确定且不崩"的块消失单点。给主控候选 A/B（SizedBox/entry-null 配方，装机实测）+ 兜底 C。

**建议主控测试顺序**：先装 Task2 的 0xbd2e1c overlay patch（确定性高，顺便看 banner 是否同源消失）→ 再在 reader 实测候选 B（0xc3def0 entry-null）→ 若无效用候选 A（a11y 配合二分定位块容器）→ 兜底 C。
