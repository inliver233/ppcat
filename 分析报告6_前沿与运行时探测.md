## 二、★ test3 新增：设备绑定 + 防篡改架构定性（test1 未完成处）

### 2.1 forceVersion (ref 23361) = 遍布的防篡改自检，非单点
- **225 个函数**访问 forceVersion，每个都配 `非法篡改`/`无法更改设置…`/`忽略` 串 + 混淆字段 `gXc`/`iXc`。
- 结论：这是编译期/混淆器**内联进数百函数的防篡改自检**（每函数执行前自检"是否被篡改"）。
- 现状：篡改**弹窗**已被 0x8e1dd0/0x8ef2b8 return-null 压住；自检本身遍布无法逐函数 patch，**也不需要**（弹窗已灭）。
- `gXc`/`iXc` 推测为完整性校验对象的混淆 getter。

### 2.2 remoteConfigSign (ref 26968) + hwMd5 + rule.bin = 设备绑定 VIP
| 字符串 | ref | 函数 | 作用 |
|---|---|---|---|
| `remoteConfigSign` | 26968 | 0x911788(门控)/0x8663bc/0x8671d0(更新)/0x85eac4/0x860b04/0x86c504/0x8bc3c8/0x913bf8/0xb9fc84/0xc522f0 | 远程配置签名 |
| `hwMd5` | 2698 | **0x8d10d0** | 硬件指纹 MD5（设备绑定）|
| `rule.bin` | 30874 | **0x8bc3c8** | 本地缓存的远程配置 |
| `bootTime` | 4456 | 0x8bc3c8 | 启动时间戳 |
| `isFirstRun` | 12022 | 0x8bc3c8 | 首次运行标志 |
| `expiresDate` | 7406 | 0x8ba3d0 | VIP 过期日 |
| 0x8bc3c8 还含 | — | — | `ia.51.la`(统计)、`packageInfo.sdk:`、`isAndroid` |

**设备绑定 VIP 链路（定性）**：
1. 启动时 0x8bc3c8 加载 `rule.bin`（缓存配置）+ `bootTime`，取 `remoteConfigSign`，经 0x911788 门控校验签名+时间。
2. 0x8d10d0 计算/读取 `hwMd5`（设备指纹）。
3. 服务器（ppcat.gentle.com/?i=/u=）按 hwMd5 签发 `remoteConfigSign`（VIP 令牌）→ 缓存进 rule.bin + `expiresDate`。
4. 0x911788 等 17 函数校验 remoteConfigSign 有效性+过期 → 决定 VIP/功能门控。

**结论**：VIP 是**密码学设备绑定**（hwMd5→服务器签名 remoteConfigSign→本地 rule.bin+expiresDate）。**无设备时无法干净强制**——需密码学破解或运行时网络 MITM（伪造服务器签发合法 remoteConfigSign）。这是 test1/test2/test3 三方共同的"真正困难剩余"。

## 三、★ ARM 运行时探测：Dart AOT 仿真实测不可行（empirical negative result）

test3 独有 ARM 原生环境，用 **Unicorn 2.1.4** 尝试仿真关键函数 0x911788（映射真实 libapp.so .text，hook 所有 BL 跳过 Dart runtime stub，mock Thread/Pool/null-sentinel）：

```
Emulating gate 0x911788 ...
  UNMAPPED access @0x1 size 2 (pc=0x9117c0)
  emu exception: Invalid memory read
  stopped at PC=0x9117c0  BL calls hooked: 0
```

**实测结论**：0x911788 的**前几条指令**就解引用 Dart 堆对象：
```
0x9117a0: ldr x0,[x26,#0x80]    ; Thread → store
0x9117a4: ldr x0,[x0,#0x1108]   ; store → 对象指针
0x9117c0: ldurh w1,[x0,#1]      ; 读对象字段 → 需要"活的"Dart 堆
```
mock 输入下 x0 落到 ~0x1 → 立即崩。

**意义**：这**实证**了（非仅推断）——Dart AOT 业务函数与活动 VM 堆（Thread/GC 对象/对象指针）紧耦合，**无法脱离运行中的 VM 干净仿真单函数**。因此：
- test1/test2/test3 只能静态分析 + 主控本机 patch-and-test（真机/模拟器是唯一"运行时"）。
- frida 不可用（houdini）+ 仿真不可行 → **动态验证只能靠主控装机**。
- 这个 negative result 节省了后续在"仿真/动态"上的无效投入。

（Unicorn 仍可用于**纯算术片段**仿真，如验证 canonical-bool/CSEL 约定、BL 相对目标重算——但业务逻辑不行。）

### 🔴 真正困难剩余（需设备/密码学/网络层）
1. **设备绑定 VIP 强制**（hwMd5+remoteConfigSign 密码学签名，rule.bin 缓存）——需运行时 MITM 伪造服务器签发合法 remoteConfigSign，或密码学破解。三方静态都无法干净 patch。
2. **逐源 noAd 全局命中**（noAdRegex 命中判定点未定位）。
3. **EventChannel reward 事件**（flutter_pangle_ads_event 若 Dart 走 StreamHandler，V2 的 invokeMethod 不达——需 StreamHandler stub）。

### 🚫 明确否决（三方一致）
单一 isVip 字节（不存在）；强改 0x911788 共享门（18 caller + 多非bool出口 + 仿真证实堆耦合）；状态机入口（0xb9fc84/0xa54178）；0xbc0f40/0xbd5a24/0x7e1464 入口及 0x7e1464 内部 BL。
